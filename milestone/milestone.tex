\milesec{1.\ Experiments so far}
Our overarching objective is an ML-research agent that mines its own logs, diagnoses defects, synthesizes verifiable RL environments for those defects, and trains a lightweight LoRA policy that improves future harness behavior. For the milestone we narrowed the verifier domain to \emph{GPU kernel engineering} (\S2) and built \texttt{kernel-synth} (\url{https://github.com/chandrasuda/kernel-synth}, private), a trace-derived environment factory. The pipeline shallow-clones a GitHub repo, AST-walks every \texttt{.py} to enumerate \texttt{nn.Module} subclasses, and runs an Anthropic tool-use harness (\texttt{list\_files / read\_file / mark\_module / finish}) to pick a diverse, non-obvious set per repo; with no API key it falls back to an AST heuristic scoring \texttt{forward} ops, custom inits/buffers, name signals (\texttt{mamba/ssm/rotary/hyena/\ldots}), and imports of \texttt{triton / flash\_attn / mamba\_ssm / einops}. A converter then emits one self-contained RL env per chosen module: \texttt{reference.py} (class + filtered imports + transitively-lifted helper defs), \texttt{inputs.py} (auto-inferred from \texttt{\_\_init\_\_}/\texttt{forward} signatures), \texttt{solution.py}, \texttt{harness.py} (eager-vs-solution timing + \texttt{torch.allclose}), and \texttt{env.json}; a small FastAPI viewer browses the catalog. A Harbor-style ATIF~v1.7 rollout layer --- \texttt{torch.compile} as the reward ceiling, Triton-only agent tools (\texttt{read\_file / write\_file / run\_benchmark / finish}) in a workspace sandbox --- is currently \emph{in flight}.

\textbf{What worked.} The heuristic correctly down-weighted \texttt{openai/whisper} (vanilla encoder/decoder $\Rightarrow$ avg 0.21) and saturated near 1.0 on novel mechanisms (\texttt{CoPE}, \texttt{DataDependentAlibi}, \texttt{Mamba2/3}, \texttt{DSSA}) --- the right inductive bias for our reward: we don't want envs on modules where a fused kernel already exists, since the speedup ceiling collapses. \textbf{What didn't.} The first converter na\"ively copied the source file's full import block, breaking many envs via \texttt{loguru}/\texttt{einx}/etc. Fix: walk the class body for referenced names, keep only imports whose bound name is used, and transitively lift module-level helper defs (e.g.\ \texttt{exists}, \texttt{default}) until a fixed point. \texttt{CoPE}'s \texttt{reference.py} now imports cleanly and \texttt{exists()} is lifted automatically --- trace-derived envs are only useful if the reference module actually runs.

\milesec{2.\ Changes to hypothesis / objective}
\textbf{The closed-loop hypothesis is unchanged}: agents improve faster when their RL training environments are mined from real trace/code distributions than when they are hand-designed. \textbf{What narrowed} is the verifier --- ``general ML-research harness defects'' $\to$ ``GPU kernel engineering'' --- because kernels admit a cheap, fully-automatable, \emph{dense} reward (numerical equivalence + wall-clock speedup vs \texttt{torch.compile}), which is exactly what makes long-horizon RL tractable; general ML-research decisions are subjective, slow to verify, and contaminated by API/cost noise, so they are better attacked once the loop is proven on the kernel sub-domain. \textbf{Preserved}: trace-derived envs, GRPO+LoRA training, evaluation vs base / SFT / prompt-only baselines, ablations over env-derivation vs synthetic. \textbf{New}: a concrete environment factory (\texttt{kernel-synth}), an external expert baseline (\texttt{torch.compile}) defining the reward ceiling, and Triton as a much smaller action space than the full agentic one --- substantially easier to RL.

\milesec{3.\ Steps to completion}
\textbf{(1) Finish the in-flight rollout layer:} ATIF~v1.7 schemas, per-env \texttt{benchmark.py} running eager + \texttt{torch.compile} + solution under \texttt{torch.allclose}, the tool-use agent loop, three rollout modes (baseline / torch\_compile / agent), and the trajectory writer. \textbf{(2)~Generate trajectories at scale:} run Claude-Sonnet and open-source coding LLMs over the 35 envs plus a held-out 5 repos to produce $N\!\approx\!200$ ATIF trajectories with rewards in $\{-0.1\}\cup[-0.2,1.5]$. \textbf{(3)~GRPO+LoRA training} on Qwen3-1.7B or SmolLM3-3B with reward $r=\mathrm{clip}((t_e-t_s)/(t_e-t_c),-0.2,1.5)$ ($t_e,t_s,t_c$~=~eager/solution/compile latency) gated by correctness; evaluate on held-out repos vs base, prompt-only self-reflection, and SFT. \textbf{(4)~Ablations:} (a)~heuristic vs agent env-selection, (b)~trace-derived envs vs a generic Triton-toy-task curriculum (KernelBench-mini style), (c)~\texttt{torch.compile} ceiling on/off. \textbf{(5)~Final eval:} best-of-$N$ solve rate, mean reward, fraction of held-out modules reaching $r\!\geq\!0.5$, and the $t_s/t_c$ distribution.
